# C2KV 实验记录 0707-0713

本文简单整理 7.7 至 7.13 期间围绕 Tool-C2KV 的主要实验、结果和阶段性结论。

## 1. agent-llm-traces 单轮 Tool Definition 压缩

实验内容：在 `agent-llm-traces` eval split 上比较 `full`、`truncate`、训练后 `c2kv` 和未训练 `c2kv_untrained`，任务是根据当前输入和工具定义生成下一步 tool call。

| truncate | 4 | 0.1698 | 0.2830 | 0.4340 | 0.6521 | 9.4814 | 3.9845 |
| truncate | 8 | 0.1415 | 0.3019 | 0.4717 | 0.6400 | 9.3088 | 7.9277 |
| c2kv | 2 | 0.1887 | 0.3585 | 0.5189 | 0.6900 | 9.7313 | 1.9995 |

| c2kv | 8 | 0.1792 | 0.3019 | 0.4811 | 0.6275 | 9.3513 | 7.9277 |

结论：训练后的 C2KV 在 agent-llm-traces 上优于未训练压缩器，也在 2x/4x 下优于 truncate；但 full 质量仍最高，单轮端到端耗时没有明显优势。

结论：Top-3 full + rest C2KV 4x 效果最好，工具名准确率达到 full 水平附近；这说明选择性保留高相关工具 schema 是一个有效方向。

## 5. 数据集检查与泛化诊断

实验内容：检查 `agent-llm-traces`、`Toolathlon`、`Hermes` 三个数据集格式和规模，并诊断 agent-llm-traces 原 split 是否存在工具泄漏。

| 数据集 | 格式 | 规模 | 工具定义长度特征 | 当前定位 |
|---|---|---:|---|---|
| agent-llm-traces | Parquet traces/spans | 1,781 traces | 诊断中发现 toolset 泄漏明显 | 第一版 C2KV 训练集 |
| Toolathlon | JSONL trajectories | 7,116 trajectories, 108 tasks | 平均约 10.5k 粗略 token，轨迹很长 | 长任务/跨数据泛化测试 |
| Hermes | Parquet ShareGPT-style | 14,701 samples | 平均约 3.3k 粗略 token，P95 约 6.3k | 多轮 reasoning/tool-call 数据 |

agent-llm-traces split 诊断：

| Split | Train sessions | Eval sessions | Train toolsets | Eval toolsets | Toolset overlap | Tool-name overlap rate | Namespace overlap rate |
|---|---:|---:|---:|---:|---:|---:|---:|
| session_disjoint | 1096 | 122 | 98 | 18 | 16 | 1.0000 | 1.0000 |
| toolset_disjoint | 950 | 268 | 90 | 10 | 0 | 0.6307 | 1.0000 |
| namespace_disjoint_proxy | 994 | 224 | 96 | 4 | 0 | 0.5983 | 0.9091 |

结论：原 session-disjoint split 存在明显工具泄漏，早期 agent-llm-traces 上的好结果更接近 IID toolset 表现，不能直接证明通用压缩能力。

## 8. 严格 split 与三数据集统一训练

实验内容：由于 agent-llm-traces 存在工具泄漏，新增 toolset-disjoint split 训练脚本，并进一步构造三数据集统一 next-action 训练格式。

统一训练目标：

```text
Task instruction
+ Tool-C2KV candidates
+ 历史 action / observation
-> 标准化下一步 action
```

统一训练设置：

| 项目 | 设置 |
|---|---|
| 数据集 | agent-llm-traces + Toolathlon + Hermes |
| 数据源采样 | 30% / 40% / 30% |
| split | task_group_id hash，70% train / 10% eval / 20% test |
| 每条 trajectory | 每 epoch 最多采样 4-8 个 step，当前静态采样 |
| 工具格式 | 70% canonical markup，20% minified JSON，10% shuffled-key JSON |
| 历史 | 最近若干 action-observation，控制 observation 长度 |
| 训练参数 | base model 冻结，只训练 C2KV/gist 相关参数 |

训练结果：

| 指标 | 数值 |
|---|---:|
| train_runtime | 2696s |
| train_samples_per_second | 0.673 |
| train_steps_per_second | 0.021 |
| train_loss | 1.774 |
| eval_loss | 1.750 |
| epoch | 1 |

结论：统一 next-action 训练流程已经跑通，但 1 epoch 下 loss 改善不明显，需要用修正后的统一评测脚本重新评估生成质量。

## 9. 统一 next-action 初版评测问题

实验内容：对新训练的统一模型进行 `full / truncate / c2kv_untrained / c2kv` 评测。

初版结果中发现评测脚本存在重复合并问题：每个 ratio shard 内部又跑了所有 ratio，导致 `c2kv/truncate/untrained` 样本数变成 300，而 `full` 只有 100。

| 方法 | ratio | 有效样本数 | 工具名准确率 | 备注 |
|---|---:|---:|---:|---|
| full | 1 | 92 | 0.2609 | 正常样本量 |
| truncate | 2/4/8 | 276 | 0.2174 / 0.2174 / 0.1848 | 被重复合并 |
| c2kv_untrained | 2/4/8 | 273 | 0.2747 / 0.2198 / 0.2747 | 被重复合并 |
| c2kv | 2/4/8 | 273 | 0.2198 / 0.1648 / 0.1319 | 被重复合并 |

结论：该表不能用于判断 trained 与 untrained 的优劣。已经修复 `compare_modes` 和 `ratios` 默认参数，下一步需要删除旧 parts 后重新跑统一评测。

## 10. 当前阶段结论

1. 早期 agent-llm-traces 上 C2KV 效果较好，但数据 split 存在工具泄漏，不能直接说明跨工具泛化能力。
2. Toolathlon 结果显示 agent-traces 训练的 C2KV 跨数据迁移较差，尤其在长工具定义和新工具集合下明显退化。
3. Top-k full + rest C2KV 是当前最有希望的推理策略，能在 agent-traces 上接近 full。
4. 三数据集统一 next-action 训练已经跑通，但还需要修正后的统一评测结果来判断是否真正提升泛化。

## 11. 下一步

1. 重新运行修复后的 unified next-action eval，确保各方法每个 ratio 样本数一致。
2. 对统一模型分别在 agent-traces、Toolathlon、Hermes 的标准评测上复测。
3. 若 untrained 仍高于 trained，需要检查学习率、训练 epoch、target 构造噪声和是否只训练 gist 参数过弱。
4. 增加 tool-name token 加权 loss，以及成功轨迹加权训练。
